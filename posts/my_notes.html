Creating Release Tags in Github using git-flow

Notes 002 - Aug 17, 2018

This can be very useful especially when creating ruby gems/engines so the user can install your gem/engine via version.

Steps in creating release tags:

  • Make sure you are in develop branch.

    Run $ git flow release start version_x.y.z

    This will create a branch called release/version_x.y.z

  • Publish the release branch to Github

    Run $ git flow release publish version_x.y.z

    This is so that the branch gets pushed to the git servers in case of any accident with the proceeding steps.

  • Make version number updates to codebase

                          # lib/foo_bar/version.rb
                      module FooBar
                        VERSION = '0.2.0' # From '0.1.0'
                      end

    Commit these changes and push to release/version_x.y.z branch

  • Finish up the release branch. The release branch will merge this release branch to master and develop to make it ready for deployment.

    Run $ git flow release finish version_x.y.z

    When prompted to give a tag for this branch, just use it version_x.y.z. After this, push the tags to github.

    Run $ git push --tags

  • Push master and develop to github

    git checkout develop

    git push origin develop

    git checkout master

    git push origin master
