Only use CRUD actions in controllers

Notes 004 - Aug 30, 2018

Ruby on Rails

This is how DHH(creator of Ruby on Rails) do with his controller.

CRUD actions are:

  • index
  • show
  • new
  • create
  • edit
  • update
  • destroy

For example, you need to show a list of messages in a page and a list of unread messages in another page, normally people do it this way.

                  # your_app/app/controllers/messages_controller.rb
                  class MessagesController < ApplicationController
                    def index
                      @messages = Message.all
                    end

                    def unread
                      @messages = Message.where(status: :unread)
                    end
                  end

Its better to create a new controller for fetches unread messages instead. So basically if you need an action that is not within the CRUD action, its better to make a new controller for it. Better yet, use a namespaced module for it to be organized.

                  # your_app/app/controllers/messages_controller.rb
                  class MessagesController < ApplicationController
                    def index
                      @messages = Message.all
                    end
                  end

                  # your_app/app/controllers/messages/unreads_controller.rb
                  module Messages
                    class UnreadsController < ApplicationController
                      def index
                        @messages = Message.where(status: :unread)
                      end
                    end
                  end

How to Facade Pattern in Ruby on Rails

Notes 003 - Aug 17, 2018

Ruby on Rails

Facade pattern is usually used in Rails' controllers to make them thin. An example of a fat controller is using a lot of variables or instance variables to use in the view. 

                  class UsersController < ApplicationController
                    def index
                      @classmates = current_user.classroom.students
                      @school     = current_user.school
                      @lessons    = Lesson.where(grade: current_user.grade)
                      @note       = Note.new
                    end
                  end

Where you can use the facade pattern to make the controller thinner and lesser variables

                  # user_index_facade.rb
                  class UserIndexFacade
                    def initialize(user)
                      @user = user
                    end

                    def classmates
                      user.classroom.students
                    end

                    def school
                      user.school
                    end

                    def lessons
                      Lesson.where(grade: user.grade)
                    end

                    def note
                      Note.new
                    end
                  end

                  # users_controller.rb
                  class UsersController < ApplicationController
                    def index
                      @facade = UserIndexFacade.new(current_user)
                    end
                  end

And you can use it in the views like:

                  <%= @facade.classmates %>
                  <%= @facade.school %>
                  <%= @facade.lessons %>
                  <%= form_for @facade.note do %>
                  <% end %>

Creating Release Tags in Github using git-flow

Notes 002 - Aug 17, 2018

Github

This can be very useful especially when creating ruby gems/engines so the user can install your gem/engine via version. Here are the cheatsheet of git-flow

Steps in creating release tags:

  • Make sure you are in develop branch.

    Run $ git flow release start version_x.y.z

    This will create a branch called release/version_x.y.z

  • Publish the release branch to Github

    Run $ git flow release publish version_x.y.z

    This is so that the branch gets pushed to the git servers in case of any accident with the proceeding steps.

  • Make version number updates to codebase

                          # lib/foo_bar/version.rb
                      module FooBar
                        VERSION = '0.2.0' # From '0.1.0'
                      end

    Commit these changes and push to release/version_x.y.z branch

  • Finish up the release branch. The release branch will merge this release branch to master and develop to make it ready for deployment.

    Run $ git flow release finish version_x.y.z

    When prompted to give a tag for this branch, just use it version_x.y.z. After this, push the tags to github.

    Run $ git push --tags

  • Push master and develop to github

    git checkout develop

    git push origin develop

    git checkout master

    git push origin master
